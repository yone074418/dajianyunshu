# GAP-003关闭设计：专业判定契约与案例配置分层

> 决策：方案A，项目委托人于2026-06-24批准
>
> 目标：不补造专业数值，不采用疑似错式自动判分，同时使G1冻结内容与第50—91天实施计划职责一致

## 1. 问题与设计结论

GAP-003把两类不同时间层级的内容合并成同一个G1阻断：

1. G1必须冻结的专业判定契约，包括输入、单位、来源、版本、完整性、比较方向、相等边界、失败与唯一恢复、保存、日志和配置缺失处理。
2. 实施计划明确安排在第50—91天形成的案例执行配置，包括车辆具体参数、三条路线障碍值、装车容差、液压区间和绑扎点坐标。

本设计将两者分层。G1冻结判定契约和配置验收模板；案例数值继续按原计划建立，但在通过来源与版本验收前不可发布案例、不可产生确定性通过结论。

## 2. 三种方案及选择

| 方案 | 内容 | 优点 | 风险 | 结论 |
|---|---|---|---|---|
| A | 冻结判定契约，案例值按第50—91天形成；疑似错式不用于自动判分 | 不造数值，符合计划时序，教学结论可审计 | 后续案例配置必须经过发布门禁 | 采用 |
| B | 直接使用论文排版式2.16/2.17计算 | 开发输入表面完整 | 量纲疑点可能形成错误教学结论 | 拒绝 |
| C | 所有案例值均要求G1前提供 | 风险最低 | 与第50—91天计划冲突，导致工作无法按计划推进 | 拒绝 |

## 3. G1冻结范围

G1冻结以下规则，不冻结尚未制作的案例数值：

- 所有专业输入必须带来源、单位、案例版本和规则版本；缺一项返回`RULE_CONFIG_INCOMPLETE`。
- 配置缺失属于发布/配置阻断，不计学生专业错误，不进入阶段通过。
- 数值比较默认使用保存的未舍入值；展示舍入不参与判定。
- 论文明确相等边界的规则沿用论文，例如绑扎夹角`θ≤60°`通过。
- 论文未明确容差或相等边界的规则，由版本化案例配置显式给出；配置未给出时不得推断。
- 学生提交只与已发布案例真值或经确认公式结果比较；页面路由、动画和模型位置不得作为真值来源。
- 上游案例或规则版本变化使相关下游结果失效，历史日志与旧版本结果保留。
- 每条配置在发布前必须通过正常、失败、边界和非法四类样例。

## 4. 疑似错式处理

论文公式2.16/2.17以及专业规则目录已登记的2.24、2.26、2.29疑点，只作为带来源和风险说明的教学材料展示，不进入首版自动判分表达式。

圆弧与直交弯道的自动判定采用版本化案例真值：学生完成规定测量后，系统按案例配置中已发布的测量目标、允许偏差、通行结论和恢复目标校验。案例发布前必须由专业教师核实来源，由课程负责人确认教学采用。系统不得从论文疑似错式自行生成真值。

如果后续取得权威勘误，经CR新增规则版本；历史尝试继续绑定原案例和规则版本，不追溯改写。

## 5. 计划职责映射

| 内容 | 形成日 | 发布门禁 |
|---|---|---|
| 车辆与挂车组合参数 | 第50、52、53天 | 来源、单位、版本、合法范围和四类样例齐全 |
| 三条路线与障碍配置 | 第57—63天 | 三路线身份、五类障碍值、正确结论和恢复目标齐全 |
| 高度、弯道、坡度、桥梁判定配置 | 第64—69天 | 不使用疑似错式自动判分；边界样例通过 |
| 车组、液压、轴载配置 | 第71—77天 | 配置来源、单位、上限/边界与回退目标齐全 |
| 装车重心、液压反馈与容差 | 第78—84天 | 坐标系、允许区间、边界和单元样例齐全 |
| 绑扎工具、点位、夹角与布置 | 第85—91天 | 工具顺序、四点真值、命中边界、`θ≤60°`样例齐全 |

“形成日”是计划内资产/数据/功能配置工作，不得提前写成虚构的G1案例值；未过发布门禁的配置不能供学生创建实验。

## 6. 状态、权限和数据影响

- 不新增或改名15个实验生命周期状态。
- `RULE_CONFIG_INCOMPLETE`是保守错误码，不是业务状态，也不计学生业务失败。
- 配置缺失时保持当前合法阶段或最近确认保存点，不进入“阶段通过”。
- 学生只能读取已发布案例的可见教学数据；不可读取未发布配置或内部真值。
- 授权教师可以查看来源与判定证据，但查看不改变尝试生命周期。
- 继续复用Day11的案例、规则版本、阶段快照、规则检查结果和日志对象；不在本设计新增数据库字段。

## 7. 关闭与重开规则

GAP-003和BLOCK-007可在以下条件同时满足后关闭：

1. 本设计经项目委托人批准。
2. 专业规则目录、状态机、G1整改包和实施计划均同步上述分层与发布门禁。
3. 不再把第50—91天案例执行值误列为G1前置材料。
4. 疑似错式明确禁止用于首版自动判分。
5. 配置不完整仍能稳定阻断阶段通过且不计学生错误。

关闭只表示G1专业设计输入无未决语义，不表示具体案例参数已经制作或专业功能已经实现。后续任一案例配置缺来源、单位、版本、边界或样例时，阻断该案例发布；若要改变本判定契约，必须重开CR，不静默改写基线。

## 8. 验收样例

| 编号 | 输入 | 预期结果 |
|---|---|---|
| GP-AC-01 | 圆弧案例缺允许偏差 | 返回`RULE_CONFIG_INCOMPLETE`，保持当前阶段，不计学生错误 |
| GP-AC-02 | 页面尝试直接使用论文式2.16计算 | 自动判分拒绝调用该表达式，记录配置审计 |
| GP-AC-03 | 已发布路线配置且学生测量命中边界 | 按该配置唯一判定并记录版本 |
| GP-AC-04 | 配置更新后打开历史完成尝试 | 继续显示原版本结果，不追溯重算 |
| GP-AC-05 | 未发布车辆参数被深链接请求 | 拒绝且不泄露内部真值，不改变尝试状态 |
| GP-AC-06 | 绑扎夹角恰为60°且其他条件通过 | 按论文明确边界判定通过 |
| GP-AC-07 | 展示值舍入后等于边界、原值超过边界 | 使用未舍入原值判定失败 |
| GP-AC-08 | 技术资源加载失败 | 进入技术恢复路径，不写学生专业错误 |

## 9. 非目标

- 不在G1阶段制作车辆、路线、液压或绑扎点具体数值。
- 不纠正或重写论文原文。
- 不把动画、三维模型坐标或截图识别值直接作为自动判分真值。
- 不编写业务代码、SQL、缓存、队列或冲突处理方案。
- 不宣称G1正式冻结；正式G1仍由第14天任务执行。
