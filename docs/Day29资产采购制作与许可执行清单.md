# Day29 资产采购/制作与许可执行清单

> 任务：第29天：建立资产采购/制作与许可执行清单和面数/贴图预算  
> 基线依据：`docs/G1资产需求基线.md`，G1 冻结资产范围 AR-001 至 AR-019  
> 执行边界：本清单只建立采购、制作、许可、性能预算和验收追踪口径；不下载、不生成、不建模、不压缩、不导入任何真实 3D 资产。  
> 版权口径：未确认来源或许可证的资源一律标为高风险，不得作为正式资产；仅可在明确采购合同、授权文件、CC0/开源许可或自制权属记录后进入正式资产流程。

## 1. 执行原则

1. 本清单覆盖 G1 已冻结的 AR-001 至 AR-019 资产类别，不新增、不删除 G1 类别。
2. 如后续发现必须调整类别、增加类别或删除类别，必须提交 CR，不得直接修改 G1 范围。
3. 推荐来源只表示后续 Day30-Day35 可调研方向，不表示已经获得、购买、下载、制作或授权。
4. 所有正式 3D 模型优先使用 GLB；教学图标、示意图可使用 SVG/PNG/WebP，但必须保留来源和许可证记录。
5. 面数与贴图预算是浏览器首版教学可用预算，后续 Day34/Day35 需以真实文件体积和性能测试复核。
