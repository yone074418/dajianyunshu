## 1. 文档目标与依据

本文档是“大件运输虚拟仿真实验教学系统”第 1 周第 2 天的用户需求基线，供后续信息架构、低保真原型、角色权限、数据库、状态机和验收用例设计直接引用。本文只定义角色、使用场景、权限边界、数据产出和可验证结果，不定义技术实现，也不把论文未给出的规则伪装为既定需求。

需求属性统一使用以下五类：

- **论文明确要求**：论文第 2—4 章直接描述。
- **根据论文合理推导**：为实现数据隔离、流程闭环、可恢复和可验收而补充的最小规则。
- **实施计划要求**：126 天实施计划明确纳入首版的能力或验收口径。
- **论文未明确**：现有论文映射和实施计划均不能确定，必须在后续需求冻结前确认。
- **首版暂不实现**：实施计划明确排除，或属于论文研究过程而非首版运行功能。

依据：

1. `article.pdf` 第 2—4 章；本日以已逐项校对的《论文功能映射》为直接引用基线。

## 3. 学生角色定义

### 3.1 使用目标

- 理解任务背景、货物参数、车辆组合、路线勘测、车组配置、装车绑扎与运输复验知识。
- 在可提示、可纠错、可回退的环境中独立完成一次六阶段实验。
- 使关键操作、错误、提示、耗时和结果被完整记录，形成可追溯的学习证据。
- 获得当前步骤反馈和实验完成状态；最终成绩是否直接向学生展示，见待确认事项 Q-04。

来源：论文明确要求，2.1、2.2、2.3.5、3.4.1—3.4.2，论文第 10—28、45—54 页（PDF 17—35、52—61）；成绩可见性论文未明确。

### 3.2 核心需求

1. 使用学号注册、使用账号密码登录，并在刷新后维持有效会话。
2. 在实验前或实验中访问资料库，按章节学习运输安全、运输车组、方案制定和运输校核知识。
3. 严格按六阶段顺序完成实验，不能跳过未通过阶段。
4. 主动请求当前步骤提示；发生阻断性错误时获得原因和纠正方法。
5. 在业务失败后返回指定步骤修改，在技术异常后从最近已保存状态继续。
6. 本人的每次尝试、关键操作、错误、提示、耗时和结果均可追溯。

来源：论文明确要求；会话恢复、禁止跳步和技术异常恢复为根据论文合理推导及实施计划要求。

### 3.3 使用前提

- 使用受支持的桌面浏览器访问系统；首版不要求移动端适配。
- 注册场景需要可用且未重复的学号；学号格式与注册审批规则论文未明确。
- 登录后身份必须绑定唯一用户 ID 和“学生”角色。
- 进入实验前，案例、关键规则与必需资源可用；断网或资源失败时不得静默丢失已保存数据。

### 3.4 可访问功能

- 学生注册、登录、退出和本人会话恢复。
- 资料库、章节导航、图文学习内容、学习活动记录。
- 本人的实验创建、继续、六阶段操作、帮助、错误反馈、失败回退、只读回放或新建尝试。
- 本人的操作日志与结果摘要；本人最终成绩、分项成绩和扣分解释仅在成绩查看/发布规则确认后开放。

### 3.5 禁止访问的数据和功能

- 禁止访问教师端路由、教师评价录入、成绩导出和教学统计。
- 禁止读取、修改、导出其他学生的资料、实验、日志、报告、评价和成绩。
- 禁止直接修改本人或他人的原始操作日志、系统评分、教师评分及权重版本。
- 已提交实验不得篡改；需要重做时创建新的实验尝试并保留原尝试。

前三项为根据论文合理推导及实施计划权限要求；已提交只读与新建尝试为实施计划要求（第 96 天）。

### 3.6 典型场景、正常路径与异常恢复

典型场景为 STU-001 至 STU-006。正常主路径：注册/登录 → 实验前知识学习 → 创建或继续本人实验 → 按顺序完成六阶段 → 系统记录日志 → 实验完成 → 评价系统形成客观评价并在教师评价齐全后形成综合成绩。错误与恢复包括凭据错误重输、错误操作查看提示后修正、路线或轴载失败回退、刷新/短暂断网后从保存点恢复。

### 3.7 需要记录的操作数据

- 用户 ID、学号、角色、会话开始/结束时间。
- 尝试 ID、案例 ID、开始/结束时间、状态、当前阶段、实验次数。
- 阶段、动作、对象、输入、判断、反馈、发生时间、结果。
- 错误类型、规则 ID、错误次数、主动/自动提示类型与提示次数。
- 测量值、单位、计算值、路线结论、配车/车组/装车/绑扎/运输结果。
- 资料章节、进入/退出时间；笔记、报告和交流数据按 26 项指标最小取数范围记录。

来源：论文明确要求，映射§4.4、§9、§10；字段命名为根据论文合理推导。

### 3.8 学生角色验收标准

- 使用两个不同学生账号验证：任一学生只能读取本人尝试、日志和获准展示的成绩数据。
- 学生访问教师端或替换资源 ID 访问他人数据时，系统返回拒绝结果或重定向，且不泄露目标数据。
- 学生可从登录连续完成资料学习和六阶段实验；每阶段顺序、继续条件、日志和失败回退均与本文一致。
- 主动提示和阻断性错误均可复现、可记录、可恢复；刷新后已提交阶段不丢失。
- 学生完成实验后形成“已完成”尝试和评分所需数据；最终成绩的学生可见性在 Q-04 确认前不作为首版学生端通过条件。

## 4. 教师角色定义

### 4.1 使用目标

- 查看授权范围内学生及其实验完成状态。
- 从单次实验详情和时间线还原学生关键决策、错误、提示和耗时。
- 对 A1—A8、B4、D4、D6、D7 共 12 项主观指标评分。
- 查看 14 项系统评价、26 项分项、四维得分、综合成绩和薄弱模块，形成教学调整依据。
- 按授权范围导出成绩用于教学归档。

### 4.2 核心需求与使用前提

- 教师必须以“教师”角色完成认证；教师账号创建方式论文未明确。
- 查看学生前必须存在明确授权范围；班级、课程和教师—学生归属模型论文未明确。
- 教师评价前，目标学生需有可评价的实验、学习或报告材料；缺项必须明确显示，不得静默按 0 分。异步学习和报告入口按PCR-006使用LRN-005/006与TEA-008/009。
- 导出前必须完成权限校验，并显示筛选范围和导出字段。

### 4.3 可访问功能与学生数据范围

- 可访问授权范围内的学生列表、完成状态、单次实验详情、原始日志只读视图、教师评价、系统评价、综合成绩、薄弱模块和成绩导出。
- “授权范围内”是强制边界，但授权依据究竟为班级、课程、教学任务还是人工分配，论文未明确，见 Q-02、Q-03。
- 教师不得查看未授权学生，也不得直接修改或删除学生原始操作日志。

### 4.4 教师评价职责

- 对 A1—A8、B4、D4、D6、D7 共 12 项逐项选择 1—5 分并可填写评语。
- 首版暂按 5＝非常优秀、4＝优秀、3＝良好、2＝及格、1＝不及格；论文表述存在方向冲突，需课程负责人确认。
- 保存时校验 12 项完整性、取值范围，并记录评分人和评分时间。
- 修改教师评分后必须触发综合成绩重新计算；修改/复核/发布流程论文未明确。

来源：论文明确要求，4.3.3，论文第 66—67 页（PDF 73—74）；完整性、审计与重算为合理推导/实施计划要求。

### 4.5 系统评价查看职责

- 查看 B1、B2、B3、B5、B6、C1—C5、D1、D2、D3、D5 共 14 项的原始值、命中区间、1—5 分、权重和数据来源。
- 发现缺失日志或区间冲突时，停止最终确认并反馈数据问题；不得用主观数值覆盖客观原始日志。
- 系统评价区间端点和 0 次活动得分未确认前，相关结果必须标记“规则待确认”，不得静默猜测。

### 4.6 典型场景、正常路径与异常恢复

典型场景为 TEA-001 至 TEA-006。正常主路径：教师登录 → 查看授权学生列表 → 选择学生与实验尝试 → 查看日志和 14 项系统评价 → 填写 12 项教师评价 → 触发并查看综合成绩/薄弱模块 → 导出授权范围内成绩。失败时按权限不足、数据缺失、评分校验失败、计算失败或导出失败分别恢复，不修改原始日志。

### 4.7 教师角色验收标准

- 教师只能看到授权范围内学生；无授权或越权资源 ID 均被拒绝。
- 单次实验详情按时间顺序展示六阶段关键输入、判断、错误、提示、耗时和结果，原始日志为只读。
- 12 项教师评价逐项齐全、范围有效才可提交；保存后记录评分人和时间。
- 14 项系统评价逐项展示原始值、区间、得分和权重；缺失或冲突有明确状态。
- 教师评分修改后综合成绩重新计算，且新结果可追溯到评分版本。
- 成绩导出只包含当前授权与筛选范围，中文不乱码，字段至少含学生标识、尝试、分项、总分和评价时间。

## 5. 评价系统职责

评价系统是后台计算主体，不是可登录的自然人角色，其职责如下：

1. 从不可被教师或学生直接改写的操作日志及学习活动记录中聚合 14 项客观指标原始值。
2. 依据唯一命中的版本化区间将原始值转换为 1—5 分；边界冲突、0 次活动和统计周期等未决规则不得自动猜测。
3. 接收 12 项教师评价，检查 26 项数据完整性。
4. 按 `G = 20 × Σ(W_i × g_i)` 计算百分制综合成绩，并保存原始值、分档、权重版本、单项贡献、四维得分和总分。
5. 当教师评分被合法修改时重新计算，不回写或删除原始日志。
6. 输出可解释的成绩和薄弱模块依据；学生能否直接查看最终成绩取决于待确认的发布规则。

来源：论文明确要求，4.3.2—4.3.4，论文第 66—69 页（PDF 73—76）；完整性、版本和不可覆盖原始日志为根据论文合理推导。

## 7. 学生端完整使用路径

### STU-001 学生注册与登录

- **触发条件**：学生首次访问系统，或已有账户需要进入系统。
- **前置条件**：浏览器可访问系统；注册时学号未被占用，登录时账户已存在。
- **使用角色**：学生。
- **输入**：学号、密码；密码复杂度与审批规则论文未明确。
- **操作步骤**：选择注册 → 输入学号和密码 → 提交 → 注册成功后进入登录 → 输入凭据 → 提交登录。
- **系统判断**：必填非空、学号唯一；登录凭据是否与用户记录匹配；账户状态规则论文未明确。
- **成功结果**：建立绑定学生用户 ID 与角色的会话，进入学生端；刷新可恢复有效会话为实施计划要求。
- **失败结果**：空值、重复学号或凭据不匹配时拒绝操作并显示明确错误，不创建错误身份会话。
- **恢复方式**：修正输入后重试；密码找回与锁定流程未明确，不提供推测路径。
- **产生的数据**：用户记录、角色、注册时间、登录会话、登录时间；不得记录明文密码。
- **后续步骤**：进入知识学习或创建/继续本人实验。
- **验收标准**：合法新学号可注册；重复/空值被拒绝；正确凭据进入学生端；错误凭据不泄露其他账户信息；学生不能进入教师端。
- **需求来源**：论文明确要求，2.2、2.3.5、3.4.1，论文第 12—13、24、45—46 页（PDF 19—20、31、52—53）；会话恢复与角色路由为实施计划要求/合理推导。

### STU-002 学生实验前知识学习

- **触发条件**：学生登录后选择“资料库/知识学习”，或实验中打开资料库。
- **前置条件**：学生会话有效；资料章节已发布。
- **使用角色**：学生。
- **输入**：知识主题或章节；可选笔记内容。
- **操作步骤**：打开资料库 → 查看目录 → 选择运输安全、运输车组、方案制定或运输校核章节 → 阅读图文内容 → 可记录笔记 → 返回。
- **系统判断**：目标章节是否存在、资源是否可加载；返回实验时是否存在可恢复的当前状态。
- **成功结果**：展示目标章节，记录学习活动；从实验中打开时返回原步骤且不丢状态。
- **失败结果**：章节或资源加载失败时显示明确错误，不把失败时长伪记为有效学习。
- **恢复方式**：重试加载、返回目录或回到最近可用内容；已保存实验状态保持不变。
- **产生的数据**：学生 ID、章节、进入/退出时间、学习时长、笔记次数及内容（若使用）。
- **后续步骤**：继续学习、返回实验或创建实验尝试。
- **验收标准**：四类主题均有可读入口；目录准确跳转；实验任一可交互阶段可打开并原状态返回；学习记录可按本人追溯。
- **需求来源**：论文明确要求，2.2、2.3.5、3.4.1，论文第 13、24、46 页（PDF 20、31、53）；进度保存为实施计划第 27 天要求。

### STU-003 学生完成六阶段仿真实验

- **触发条件**：学生选择创建新实验或继续未完成尝试。
- **前置条件**：学生已登录；唯一首版案例、三条路线、规则与必要资源可用；继续时尝试归属本人。
- **使用角色**：学生。
- **输入**：货物/任务确认、配车参数、路线测量与选择、车组配置、装车位置、绑扎工具/点位/角度、最终路线复验选择。
- **操作步骤**：依次完成 ①运输任务及货物介绍 → ②简单配车 → ③路线勘测 → ④车组确定 → ⑤货物装车与绑扎加固 → ⑥货物运输；每阶段通过后保存并进入下一阶段。
- **系统判断**：当前尝试归属、阶段顺序、必填输入、专业教学规则、继续条件、资源和保存状态；任何未通过阶段不得跳过。
- **成功结果**：六阶段全部通过，正确路线运输动画完成，尝试状态变为“已完成”，日志和评分输入完整。
- **失败结果**：规则失败阻止继续并给出原因；运输路线失败播放对应反馈并返回路线选择；技术失败不把尝试错误标记为已完成。
- **恢复方式**：按错误指向返回当前或指定前序步骤修正；刷新/短断网后从最近已保存阶段继续；已提交尝试重做时创建新尝试。
- **产生的数据**：尝试、阶段状态、所有关键输入/判断/反馈、错误、提示、耗时、结果和完成时间。
- **后续步骤**：评价系统生成 14 项客观评价；教师完成 12 项主观评价；系统计算综合成绩。
- **验收标准**：端到端顺序与本文完全一致；任一阶段未通过不能进入下一阶段；正确路径完成，错误路径可恢复；每个 P0 提交动作至少一条可追溯日志。
- **需求来源**：论文明确要求，2.3.5、3.4.2，论文第 25—28、48—54 页（PDF 32—35、55—61）；保存恢复为实施计划 G5。

### STU-004 学生发生错误并使用操作提示

- **触发条件**：学生主动点击“帮助”，或关键操作命中阻断性错误规则。
- **前置条件**：学生处于本人进行中的实验和可提示步骤。
- **使用角色**：学生。
- **输入**：当前阶段、动作、对象、错误上下文；主动帮助无业务输入。
- **操作步骤**：执行错误操作或点击帮助 → 阅读当前知识点/错误原因与纠正方法 → 关闭提示 → 修正操作 → 重新提交。
- **系统判断**：提示是否与当前步骤匹配；错误是否阻断；同一规则是否为再次错误并需要升级提示。
- **成功结果**：提示不直接代做，学生修正后通过；提示使用和错误均写入日志。
- **失败结果**：提示加载失败或修正后仍不合规则继续阻止后续；不得因提示失败跳过规则。
- **恢复方式**：重试提示、查阅资料库、重置当前步骤或再次修正；第二次同类错误显示更具体方法但不自动完成。
- **产生的数据**：规则 ID、错误类型/次数、提示类型/次数、提示时间、修正动作和结果。
- **后续步骤**：继续当前阶段或按规则回退。
- **验收标准**：主动帮助可显示/关闭；每个 P0 规则失败有明确、可执行反馈；同一规则二次失败提示升级；所有事件顺序可追溯。
- **需求来源**：论文明确要求，2.2、2.3.5、3.4.1，论文第 13、24—25、47 页（PDF 20、31—32、54）；不自动代做为合理推导。

### STU-005 学生实验失败后回退重试

- **触发条件**：轴线载荷超限、路线复验失败，或其他阶段规则明确要求回退。
- **前置条件**：存在本人进行中的尝试及最近保存点。
- **使用角色**：学生。
- **输入**：失败规则、失败阶段和已有配置。
- **操作步骤**：查看失败原因 → 选择系统提供的返回入口 → 回到指定阶段 → 修改参数或选择 → 重新校核 → 通过后继续。
- **系统判断**：失败类型对应的合法回退目标；修改后是否必须重算；旧结果是否已失效。
- **成功结果**：返回正确步骤，保留历史日志，修改后的新判断通过并恢复主流程。
- **失败结果**：再次不满足规则则继续阻止；非法跳步被拒绝；保存失败时不覆盖最后有效保存点。
- **恢复方式**：继续修改重试、重置当前步骤，或退出后从最后保存点继续。
- **产生的数据**：失败原因、回退目标、修改前后输入、重试次数、新判断和时间。
- **后续步骤**：进入通过后的下一阶段；运输失败返回路线选择，轴载失败返回挂车参数选择。
- **验收标准**：至少验证轴载超限和错误路线两类回退；回退目标正确，历史失败日志保留，新结果重新计算，刷新后状态一致。
- **需求来源**：论文明确要求，3.4.2，论文第 53—54 页（PDF 60—61）；通用保存恢复为实施计划 G5。

### STU-006 学生完成实验并形成成绩

- **触发条件**：一次实验尝试状态变为“已完成”。
- **前置条件**：六阶段成功日志完整；评价规则和权重版本有效；综合成绩还需 12 项教师评价齐全。
- **使用角色**：学生（完成行为）、评价系统（计算）、教师（补充评价）。
- **输入**：实验/学习原始数据、14 项系统评价输入、12 项教师评分、26 项权重。
- **操作步骤**：学生完成实验 → 系统聚合客观数据 → 生成 14 项系统评价 → 教师提交 12 项评价 → 系统检查完整性并计算综合成绩。
- **系统判断**：数据是否齐全、每项分档是否唯一、权重版本是否有效；学生成绩查看权限是否已按待确认发布规则开放。
- **成功结果**：保存 26 项分值、四维得分、单项贡献、权重版本和百分制总分；学生至少可见实验完成状态，最终成绩是否可见待确认。
- **失败结果**：缺项、区间冲突或权重无效时不生成貌似有效的总分，而显示缺失/冲突清单。
- **恢复方式**：补齐教师评价、修正规则配置或恢复缺失日志后重新计算；不得手工改总分绕过来源数据。
- **产生的数据**：14 项客观原始值/分档、12 项教师评分、26 项贡献、四维得分、总分、版本和计算时间。
- **后续步骤**：教师查看诊断和导出；学生查看最终成绩须待 Q-04/Q-05 确认。
- **验收标准**：固定样例手算结果与系统公式一致；缺项不按 0 静默计算；任一分项可追溯；学生未获发布权限时不能通过接口读取最终成绩。
- **需求来源**：论文明确要求，4.3.2—4.3.4，论文第 66—69 页（PDF 73—76）；学生可见性论文未明确。

## 8. 教师端完整使用路径

### TEA-001 教师查看学生列表

- **触发条件**：教师登录后进入学生管理。
- **前置条件**：教师身份有效且已配置学生查看授权范围；授权模型论文未明确。
- **使用角色**：教师。
- **输入**：姓名、班级、完成状态等筛选条件；班级归属论文未明确。
- **操作步骤**：进入学生列表 → 输入搜索/筛选 → 查看结果 → 选择一个学生。
- **系统判断**：教师是否有权查看目标学生；筛选条件是否有效。
- **成功结果**：只返回授权范围内学生及其实验状态。
- **失败结果**：无权限、无数据或查询失败时显示对应状态，不混同为空列表。
- **恢复方式**：调整筛选、重试查询；授权缺失时联系支持人员，不能自行扩大权限。
- **产生的数据**：查询条件、结果数量和访问审计；不改变学生数据。
- **后续步骤**：进入选定学生的单次实验详情。
- **验收标准**：姓名/班级/状态筛选有效；越权学生不出现在列表且直接访问也被拒绝。
- **需求来源**：论文 4.3.4 第 69 页（PDF 76）；筛选为实施计划第 106 天要求；授权范围论文未明确。

### TEA-002 教师查看单次实验详情和操作日志

- **触发条件**：教师从授权学生列表选择一次实验尝试。
- **前置条件**：目标学生和尝试在授权范围，且存在日志。
- **使用角色**：教师。
- **输入**：学生 ID、尝试 ID。
- **操作步骤**：打开详情 → 查看尝试摘要 → 按时间线展开六阶段 → 查看关键输入、判断、反馈、错误、提示和耗时。
- **系统判断**：资源归属、教师权限、日志完整性与时间排序。
- **成功结果**：教师可从头到尾还原关键决策过程，原始日志只读。
- **失败结果**：越权被拒绝；日志缺失明确标记缺口；加载失败不显示不完整数据为“完整”。
- **恢复方式**：重试加载；对缺失日志发起数据问题记录；不得补写或修改原始日志。
- **产生的数据**：访问审计；不产生对原日志的修改。
- **后续步骤**：查看系统评价或填写教师评价。
- **验收标准**：六阶段时间线顺序稳定；每阶段关键输入、判断、反馈可定位；界面和接口均无日志修改能力。
- **需求来源**：论文明确要求，2.2、3.4.1、4.3.4，论文第 13、47、69 页（PDF 20、54、76）。

### TEA-003 教师填写 12 项主观评价

- **触发条件**：教师在授权学生实验详情中选择“教师评价”。
- **前置条件**：教师有评价权限；目标材料可查看；12 项量表已加载。
- **使用角色**：教师。
- **输入**：A1—A8、B4、D4、D6、D7 各 1—5 分，可选评语。
- **操作步骤**：逐项查看指标和材料 → 选择分值 → 填写评语 → 提交保存；需要修改时重新编辑并保存。
- **系统判断**：12 项是否齐全、分值是否在 1—5、评分目标和评分人是否有效。
- **成功结果**：保存评分人、时间和 12 项分值；新增或修改评分触发综合成绩重算。
- **失败结果**：缺项、越界、越权或保存失败时不形成完整评价，不触发错误总分。
- **恢复方式**：定位缺项/非法项后修正；保存失败重试并避免重复版本。
- **产生的数据**：教师评价明细、评语、评分人、评分时间、版本/修改时间。
- **后续步骤**：查看 14 项系统评价和重算后的综合成绩。
- **验收标准**：12 项名称与论文最终指标一致；缺一项不能按“完整”提交；越界拒绝；修改后总分重算且历史依据可追溯。
- **需求来源**：论文明确要求，4.3.3—4.3.4，论文第 66—69 页（PDF 73—76）；5 分最好为合理推导，待确认。

### TEA-004 教师查看 14 项系统评价

- **触发条件**：教师打开目标尝试的系统评价页。
- **前置条件**：目标在授权范围；客观日志聚合已执行。
- **使用角色**：教师。
- **输入**：学生 ID、尝试 ID。
- **操作步骤**：打开系统评价 → 逐项查看 B1、B2、B3、B5、B6、C1—C5、D1、D2、D3、D5 → 展开原始值、区间、分值和权重。
- **系统判断**：权限、原始数据是否齐全、每项是否唯一命中区间、采用的规则版本。
- **成功结果**：14 项依据完整可解释，教师以只读方式查看。
- **失败结果**：缺失、区间冲突或规则未确认时显示明确状态，不生成伪确定分值。
- **恢复方式**：重试聚合或交由支持人员修复数据/配置；教师不得人工覆盖原始值。
- **产生的数据**：查看审计；若重新聚合则产生带版本的评价结果。
- **后续步骤**：完成教师评价或查看综合成绩。
- **验收标准**：14 项不多不少；每项显示原始值、命中区间、1—5 分、权重和来源；冲突项不能进入最终计算。
- **需求来源**：论文明确要求，4.3.3—4.3.4，论文第 68—69 页（PDF 75—76）；边界待确认见 Q-09。

### TEA-005 教师查看综合成绩和薄弱模块

- **触发条件**：26 项评价齐全且系统已计算综合成绩。
- **前置条件**：教师有目标学生权限；有效权重版本存在。
- **使用角色**：教师。
- **输入**：学生 ID、尝试 ID 或成绩记录 ID。
- **操作步骤**：打开成绩详情 → 查看总分 → 查看 A/B/C/D 四维得分和 26 项贡献 → 查看低分指标及对应错误模块/日志证据。
- **系统判断**：成绩是否使用完整数据和有效版本；薄弱点是否有可追溯依据。
- **成功结果**：显示可复算总分、四维分、薄弱指标/模块及证据入口。
- **失败结果**：数据不完整或计算失败时显示原因，不显示旧结果为当前结果。
- **恢复方式**：补齐评价、重新计算或修复配置后刷新；不得直接编辑总分。
- **产生的数据**：查看审计；不修改原始成绩，重算时生成新的计算版本/时间。
- **后续步骤**：教学反馈、调整教学或导出成绩。
- **验收标准**：固定样例可按公式复算；薄弱模块至少能追溯到分项和相关日志；教师评分修改后页面显示重算结果。
- **需求来源**：论文明确要求，2.1、4.3.4，论文第 12、69 页（PDF 19、76）；诊断展示口径为合理推导。

### TEA-006 教师导出成绩

- **触发条件**：教师在学生列表或成绩页面选择导出。
- **前置条件**：教师身份有效；筛选结果均在授权范围；存在可导出的成绩。
- **使用角色**：教师。
- **输入**：授权范围、筛选条件、导出格式；首版至少 CSV，打印视图按计划实现。
- **操作步骤**：设置筛选 → 预览数量/字段 → 确认导出 → 获取文件。
- **系统判断**：每条记录的授权、字段完整性、导出编码和生成状态。
- **成功结果**：生成仅包含授权范围的成绩文件，中文不乱码，字段含学生标识、尝试、分项、总分和评价时间。
- **失败结果**：越权记录被拒绝且整个导出给出明确错误；生成失败不返回损坏文件。
- **恢复方式**：缩小范围、修正筛选或重试生成；权限问题不得通过导出绕过。
- **产生的数据**：导出任务、筛选条件、记录数量、发起人、时间和结果审计。
- **后续步骤**：教学归档或线下分析；成绩发布不由导出自动触发。
- **验收标准**：导出行数与授权筛选结果一致；中文 UTF-8 可正确打开；无未授权学生；必需字段完整。
- **需求来源**：实施计划第 110 天；论文第 2—4 章未明确，属性为实施计划要求/根据论文合理推导。

## 14. 首版范围与排除范围

### 14.1 首版范围

- 学生端、教师端、身份与权限、知识学习、提示、日志、六阶段完整闭环。

来源：论文明确要求与实施计划“本期必须做”。

### 14.2 首版排除范围

- VR/头显/数据手套、多人同步实验、即时聊天、移动端适配和开放世界。

- 多案例编辑器、排行榜、AI 辅导、影视级模型与特效。
- 德尔菲专家问卷管理和 AHP 现场计算工具；首版使用论文既定指标/权重的版本化配置。
- 未确认的管理员独立业务端、成绩发布/复核工作流、密码找回/锁定，不因“暂未确认”而暗中实现默认规则。

来源：实施计划“严格范围”和映射§14；属性为首版暂不实现或论文未明确。
