## 3. 签署要求

每项材料必须包含：采用值/公式、来源页或原始文件、变量定义、单位、精度/容差、等号边界、适用案例与规则版本、正常/失败/边界/非法样例，以及专业教师核实和课程负责人确认。材料未齐时，状态机只能返回规则配置不完整或阻断表达；技术异常不得代替业务失败。

## 4. 审计结论

现有仓库可以证明论文给出了部分公式与截图示例，但不能证明首版案例的全部确定性专业参数已经冻结。GAP-003、BLOCK-007及ACT-009不能诚实关闭，正式G1不能通过，也不应在此条件下把整改分支合入`main`。
